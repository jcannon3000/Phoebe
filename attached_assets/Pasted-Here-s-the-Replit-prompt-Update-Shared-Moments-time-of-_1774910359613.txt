THE CORE CHANGE
Spiritual practice Shared Moments do not have a fixed clock time and a one-hour window. Instead they have a time of day. Each member chooses what that time of day means for their own schedule. The calendar is personal — Eleanor puts events on each person’s calendar at the time that works for them, not a synchronized clock time.
This reflects how spiritual practice actually works. Morning Prayer is a morning practice. When exactly in the morning is between you and your calendar.

ROLLING CALENDAR SYSTEM — TWO EVENTS IN ADVANCE
Eleanor never puts the full recurring series on anyone’s calendar. Instead Eleanor maintains exactly two upcoming calendar events per member per practice at all times.
The rule:
At any given moment each member has exactly two upcoming Eleanor calendar events for each spiritual practice they belong to.
When a member logs their practice Eleanor immediately adds a new event further in advance so the count returns to two.
When a member joins Eleanor immediately adds their next two upcoming events.
How this works technically:
Store each calendar event Eleanor creates:

moment_calendar_events
  id uuid PK
  shared_moment_id uuid FK shared_moments
  moment_member_id uuid FK moment_members
  google_calendar_event_id text
  scheduled_for timestamptz
  logged boolean default false
  logged_at timestamptz
  created_at timestamptz


On member joining:
Calculate the next two occurrences of the practice based on their personal_time and the practice frequency.
Create two Google Calendar events — one for the next occurrence and one for the one after.
On member logging:
Mark the current event as logged.
Calculate the next occurrence after the furthest existing event.
Create one new calendar event for that occurrence.
Total count of upcoming events returns to two.
If a member has not logged and a scheduled event passes:
Mark it as missed.
Do not automatically add a replacement — the count drops to one.
When they next log, restore to two.
This keeps the calendar clean. Members never see a months-long recurring series stretching into the future. They see the next practice and the one after. Nothing more.

STREAK LOGIC UPDATE
A day counts toward the group streak if at least two members log on that practice day in their respective timezones.
Because members are in different timezones and have different personal times the streak window is the full calendar day in each member’s timezone. Eleanor checks at the end of each day whether the threshold was met.
Store streak evaluation results:

moment_streak_days
  id uuid PK
  shared_moment_id uuid FK shared_moments
  practice_date date
  members_logged integer
  bloomed boolean
  evaluated_at timestamptz


Evaluate at midnight UTC each day for all practices that had a scheduled occurrence that day.
If bloomed: increment consecutive_windows, update milestone_state.
If missed: apply existing miss rules.
